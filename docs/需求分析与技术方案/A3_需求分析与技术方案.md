## 二、需求分析

### 2.1 业务背景与问题定义

目标场景：高等教育个性化学习辅助。

核心痛点：
- 学习资源繁杂无序，难以精准匹配个人需求
- 课堂集体讲授无法兼顾每位学生的学习节奏
- 传统系统缺乏多模态生成、多智能体协同能力
- 学生知识掌握差距大，缺乏个性化指导

系统目标：以某一具体专业课程为切入点，构建多智能体系统，实现个性化资源自动生成、学习路径规划和精准资源推送。

**推荐课程选择：人工智能导论 / 机器学习**

- 团队最熟悉，资料丰富
- 知识点结构清晰，适合生成各类资源
- 代码案例、思维导图、练习题都好做

### 2.2 功能需求分析

#### 功能1：对话式学习画像构建（必做）

交互方式：自然语言对话（非表单填写）

画像维度（≥6个，建议8个）：

| 维度 | 说明 | 采集方式 |
|------|------|----------|
| 知识基础 | 已掌握的前置知识 | 对话提问 |
| 学习目标 | 期望达到的学习效果 | 对话提问 |
| 认知风格 | 偏好图文/代码/文字 | 对话推断 |
| 易错点偏好 | 历史上容易出错的知识点 | 对话提问 |
| 学习进度 | 当前学到哪个章节 | 对话提问 |
| 时间投入 | 每天可用学习时间 | 对话提问 |
| 专业背景 | 所学专业方向 | 对话提问 |
| 学习历史 | 已完成的练习/资源 | 系统记录 |

关键子需求：
- 画像从对话中自动抽取（LLM 结构化提取）
- 画像持久化存储（PostgreSQL）
- 画像随学习进展动态更新
- 画像可视化展示（雷达图或卡片）

#### 功能2：多智能体协同资源生成（必做，核心）

必须体现"多智能体"架构，至少 5 种资源类型。

**推荐的 5 种资源（稳定可实现）：**

| 资源类型 | 生成方式 | 展示形式 |
|----------|----------|----------|
| 知识讲解文档 | LLM 生成 Markdown | Markdown 渲染 |
| 知识点思维导图 | LLM 生成 JSON → ECharts/Mermaid 渲染 | 交互式图表 |
| 分层练习题 | LLM 生成（选择/填空/简答） | 卡片式题目 |
| 代码实操案例 | LLM 生成 Python 代码 + 注释 | 代码高亮块 |
| 章节要点总结卡片 | LLM 生成结构化摘要 | 卡片组件 |

**可选第6种（加分）：**
- 拓展阅读材料（RAG 检索 + LLM 整理）

**暂不建议：**
- 多模态视频/动画（生成成本高，演示不稳定）

关键子需求：
- 每种资源由独立 Agent 负责生成
- Agent 之间通过 LangGraph 状态机协调
- 资源生成支持流式输出（避免白屏等待）
- 生成进度可追踪

#### 功能3：个性化学习路径规划与资源推送（必做）

输入：学生画像 + 课程知识图谱
输出：

- 个性化学习路径（有序知识点序列）
- 每个知识点对应的推荐资源列表
- 学习步骤说明

关键子需求：
- 知识点依赖关系图（课程知识图谱）
- 基于画像的路径规划算法（LLM 辅助）
- 资源推送（从已生成资源库中匹配）
- 路径可视化（时间线或树形图）

#### 功能4：智能辅导（可选加分项）

- 学生提问 → 多模态答疑（文字 + 图解）
- 基于知识库的 RAG 问答
- 针对性学习引导

#### 功能5：学习效果评估（可选加分项）

- 跟踪练习完成情况
- 多维度评估（正确率、完成度、时间投入）
- 动态调整推送策略

### 2.3 非功能需求

| 需求项 | 具体要求 |
|--------|----------|
| 流式输出 | LLM 生成内容实时流式展示，避免白屏 |
| Markdown 渲染 | 所有文档类资源支持 Markdown 渲染 |
| 卡片化展示 | 多模态内容以卡片形式组织 |
| 防幻觉 | 生成内容需有事实校验机制 |
| 内容安全 | 过滤敏感/违规内容 |
| 响应时间 | 核心功能响应合理，提供生成进度追踪 |
| 界面 | 符合现代 AI 产品交互规范 |

### 2.4 评分权重分析

| 评分项 | 占比 | 对应重点 |
|--------|------|----------|
| 创新价值与实用性 | 35% | 多智能体设计、个性化效果、AI 技术融合 |
| 功能实现及技术要求 | 45% | 5种资源生成、画像6维、路径规划 |
| 配套文档 | 10% | 需求分析、技术文档完整性 |
| 演示视频与PPT | 10% | 演示效果、逻辑清晰度 |

> 功能实现占 45%，是最重要的评分项，必须保证 5 种资源全部可演示。

## 三、技术选型

### 3.1 前端

**推荐：Streamlit（主路线）**

理由：
- A3 无硬件/OS 限制，Streamlit 可直接使用
- 天然支持流式输出（`st.write_stream`）
- 支持 Markdown 渲染
- 支持自定义组件（ECharts 思维导图）
- 开发速度快，2人团队可控
- 多页面支持，适合多功能系统

关键 Streamlit 特性利用：
- `st.chat_message` / `st.chat_input`：对话式画像构建
- `st.write_stream`：流式资源生成
- `st.components.v1.html`：ECharts 思维导图嵌入
- `st.tabs` / `st.columns`：卡片化资源展示
- `st.session_state`：画像状态管理

**备选：FastAPI + React/Vue**（前后端分离，开发成本更高，不推荐作为首选）

### 3.2 后端

**推荐：FastAPI + Uvicorn**

- 提供 REST API 供 Streamlit 调用
- 处理异步任务（资源生成）
- 管理数据库连接

### 3.3 大模型

**主力 LLM：讯飞星火大模型 API**
- 出题企业是科大讯飞，使用讯飞 API 有加分可能
- 支持流式输出
- 支持 Function Calling

**备选 LLM：**
- Claude API（claude-sonnet-4-6）：推理能力强，适合复杂资源生成
- Qwen API：中文效果好，性价比高
- DeepSeek API：代码生成能力强

**建议策略：**
- 主要演示和文档中突出讯飞星火
- 实际开发中可多模型并用，取最佳效果

**Embedding 模型：**

- 讯飞 Embedding API 或 BGE-M3（本地）

### 3.4 多智能体框架

**推荐：LangGraph**

理由：
- 状态机编排，适合多步骤、多角色协作
- 支持条件分支（根据画像决定生成哪些资源）
- 支持人工干预节点
- 可视化 Agent 流程图（便于文档说明）

Agent 角色设计：

| Agent | 职责 |
|-------|------|
| ProfileAgent | 对话式画像构建与更新 |
| CurriculumAgent | 课程知识点分析与结构化 |
| PlannerAgent | 学习路径规划 |
| DocAgent | 知识讲解文档生成 |
| MindmapAgent | 思维导图生成 |
| QuizAgent | 练习题生成 |
| CodeAgent | 代码案例生成 |
| SummaryAgent | 要点总结卡片生成 |
| SafetyAgent | 内容安全与防幻觉校验 |
| RecommendAgent | 资源推送与路径更新 |

### 3.5 RAG 框架

**推荐：LangChain**

- 文档加载、分块、向量化
- 检索链路（相似度检索 + Rerank）
- 与 LangGraph 无缝集成

### 3.6 向量数据库

**推荐：Qdrant（本地模式）**

- 单文件部署，无需额外服务
- Python 客户端，易用

**备选：ChromaDB**

- 纯 Python，零配置

### 3.7 关系型数据库

**推荐：PostgreSQL**

- 存储学生画像、学习记录、生成的资源元数据

**开发阶段：SQLite**（快速启动）

### 3.8 课程知识库构建

选定课程：**人工智能导论**（建议）

知识库内容：
- 教材 PDF（如《人工智能：一种现代方法》相关章节）
- 课程讲义（自行整理）
- 知识点列表（手动构建，约 50-100 个知识点）
- 知识点依赖关系（手动标注）

知识库格式：
- 文本文档 → 向量化存入 Qdrant
- 知识点结构 → 存入 PostgreSQL
- 知识点关系图 → NetworkX + PostgreSQL

### 3.9 思维导图技术方案

LLM 生成结构化 JSON → 前端 ECharts 渲染

```json
{
  "name": "机器学习",
  "children": [
    {
      "name": "监督学习",
      "children": [
        {"name": "线性回归"},
        {"name": "决策树"}
      ]
    }
  ]
}
```

ECharts Tree 图或 Markmap 渲染。

### 3.10 技术栈汇总

| 层次 | 技术选型 | 备注 |
|------|----------|------|
| 前端 | Streamlit | 流式输出、Markdown、卡片化 |
| 后端 | FastAPI | API 服务、异步任务 |
| LLM | 讯飞星火 API（主）+ Claude/Qwen（辅） | 讯飞为主突出合规性 |
| Embedding | BGE-M3 / 讯飞 Embedding | 中文效果好 |
| Agent 编排 | *LangGraph*（**学习周期：1month**） | 多智能体状态机 |
| RAG | *LangChain* | 检索链路 |
| 向量库 | Qdrant（本地） | 轻量部署 |
| 关系库 | PostgreSQL/MySQL | 画像、记录存储 |
| 思维导图 | ECharts / Markmap | 前端渲染 |
| 知识图谱 | *NetworkX* + ECharts | 课程知识点关系 |

### 4.4 数据库完整数据模型

#### 用户与会话

```sql
-- 用户表
CREATE TABLE "user" (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username    VARCHAR(64) UNIQUE NOT NULL,
    password_hash VARCHAR(128) NOT NULL,
    email       VARCHAR(128),
    created_at  TIMESTAMPTZ DEFAULT now()
);

-- 对话会话表
-- 画像构建对话和智能辅导对话均复用此表，通过 session_type 区分
CREATE TABLE chat_session (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES "user"(id) ON DELETE CASCADE,
    session_type VARCHAR(32) NOT NULL CHECK (session_type IN ('profile', 'tutoring')),
    title       VARCHAR(128),
    messages    JSONB NOT NULL DEFAULT '[]',
    -- messages 结构: [{"role": "user"|"assistant", "content": "...", "ts": "..."}]
    created_at  TIMESTAMPTZ DEFAULT now(),
    updated_at  TIMESTAMPTZ DEFAULT now()
);
```

#### 课程与知识图谱

知识图谱采用五层节点结构，`kg_node` 和 `kg_edge` 两张表统一存储：

```
Course（课程）
  └── Chapter（章节）
        └── KnowledgePoint（知识点）
              └── SubPoint（子知识点）
                    └── Concept（概念/术语）
```

边类型定义：

| 边类型 | 方向 | 语义 |
|--------|------|------|
| `IS_PART_OF` | 子 → 父 | 从属关系，构建层级骨架 |
| `REQUIRES` | 后 → 前 | 学习后者前必须掌握前者 |
| `RELATED_TO` | 双向 | 相关但无强依赖 |
| `CONTAINS` | 知识点 → 概念 | 知识点涉及的术语/概念 |

```sql
-- 知识图谱节点表
-- Course / Chapter / KnowledgePoint / SubPoint / Concept 统一存储
CREATE TABLE kg_node (
    id          VARCHAR(64) PRIMARY KEY,   -- 如 "kp_303"
    type        VARCHAR(32) NOT NULL CHECK (type IN ('Course','Chapter','KnowledgePoint','SubPoint','Concept')),
    name        VARCHAR(128) NOT NULL,
    description TEXT,
    difficulty  SMALLINT CHECK (difficulty BETWEEN 1 AND 5),  -- 仅 KnowledgePoint/SubPoint 有效
    is_core     BOOLEAN DEFAULT false,                         -- 仅 KnowledgePoint 有效
    extra       JSONB DEFAULT '{}',        -- 扩展字段，如 definition(Concept)、order(Chapter)
    created_at  TIMESTAMPTZ DEFAULT now()
);

-- 知识图谱边表
CREATE TABLE kg_edge (
    id          BIGSERIAL PRIMARY KEY,
    source_id   VARCHAR(64) NOT NULL REFERENCES kg_node(id) ON DELETE CASCADE,
    target_id   VARCHAR(64) NOT NULL REFERENCES kg_node(id) ON DELETE CASCADE,
    relation    VARCHAR(32) NOT NULL CHECK (relation IN ('IS_PART_OF','REQUIRES','RELATED_TO','CONTAINS')),
    UNIQUE (source_id, target_id, relation)
);
CREATE INDEX idx_kg_edge_source ON kg_edge(source_id);
CREATE INDEX idx_kg_edge_target ON kg_edge(target_id);
```

各 Agent 对图谱的使用方式：

| Agent | 使用的边/字段 | 用途 |
|-------|-------------|------|
| PlannerAgent | `IS_PART_OF` + `REQUIRES` | 拓扑排序生成合法学习序列 |
| MindmapAgent | `IS_PART_OF` | 直接提取树形结构渲染思维导图 |
| QuizAgent | `difficulty` + `is_core` | `is_core=true` 出必考题，按难度分层 |
| RecommendAgent | `REQUIRES`（反向） | 知识点薄弱时找出所有受影响的后续知识点 |

### 4.5 防幻觉机制设计

1. RAG 引用溯源：生成内容必须标注知识库来源
2. 置信度过滤：检索相似度低于阈值时，提示"知识库中无相关内容"
3. SafetyAgent 校验：
   - 检查关键事实（如算法名称、公式）是否与知识库一致
   - 过滤敏感/违规内容
4. 用户反馈：提供"内容有误"标记入口，触发人工审核
